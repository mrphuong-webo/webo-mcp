# WEBO MCP — GitHub repo SEO (Agentic adapt)

Upstream: [seo-github.md](https://github.com/Bhanunamikaze/Agentic-SEO-Skill/blob/main/resources/skills/seo-github.md). Index: [`webo-mcp-seo-agentic`](../webo-mcp-seo-agentic/SKILL.md).

## Checklist (condensed)

- README H1 = repo name; summary in first 120 chars; badges; Install / Quick start / Features / Contributing / License.
- Topics: 5–10 precise GitHub topics.
- About: description + website + “Include in commits” for Releases.
- `docs/` or GitHub Pages; Open Graph / social image.
- Issues/Discussions templates; clean `good first issue` labels.

## WEBO MCP

**Does not use** `webo/*` — this is not WordPress. Use the GitHub UI, **gh** CLI, or **`github_seo.py`** in the Agentic repo if you clone it.

When the user wants a **WordPress post** introducing the repo, use [`webo-mcp-wordpress-content`](../webo-mcp-wordpress-content/SKILL.md).
